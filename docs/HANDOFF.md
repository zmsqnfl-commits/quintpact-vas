# VAS 2.6.4 AI 인계

새 프로젝트와 기존 프로그램 모두 같은 간단한 인계 흐름을 사용합니다.

```text
프롬프트 복사 → 코딩 AI에서 원본 확인 → 실제 작업
               └ VAS-AI-HANDOFF.json(v3)은 설정 보관이 필요할 때만 선택
```

## 사용

1. 기존 프로그램은 **폴더 위치**를 골라 복사 프롬프트에 포함합니다.
2. 새 프로젝트는 빈 폴더, 기존 프로그램은 프롬프트에 적힌 원본 폴더를 코딩 도구에서 엽니다.
3. VAS에서 복사한 프롬프트를 붙여넣습니다.
4. 설정 보관이 필요하면 `VAS-AI-HANDOFF.json`도 프로젝트 최상위에 저장합니다.

## JSON 계약

- `workflow`: `handoffId`, 반복 번호, 이전 결과 연결
- `project`: 이름, 신규·기존 구분, 목표
- `task`: 요청, 제약, 완료 기준
- `context.requirements`: 사용자가 확인한 요구사항
- `context.design`: 프리셋, 토큰, 디자인 지침
- `context.rag`: 기본값은 비포함
- `context.continuation`: 기본값은 비포함
- `qualityGate`: 요구사항·디자인·RAG·개인정보 확인 상태
- `security`: 원본 불변과 제외 규칙
- `assistantGuide`: 코딩 도구에 붙여넣을 문장
- `integrity.payloadSha256`: 인계 내용 연결용 SHA-256

절대 경로, 비밀값, 연락처, 작업 기억 원본은 JSON에 포함하지 않습니다. 사용자가 고른 폴더 위치는 로컬에서 복사하는 프롬프트에만 포함합니다.
`analysis`와 파일 목록은 포함하지 않으며 실제 구조는 코딩 AI가 원본 폴더에서 확인합니다.

## 호환 계약

`VAS-AI-RESULT.json` 검증과 RAG 선택 모듈은 이전 데이터 호환을 위해 남아 있지만 일반 화면과 기본 프롬프트에는 표시하지 않습니다. 기본 저장·복사 동작도 반복 작업 영수증을 만들지 않습니다.

## RAG와 RBG

- RAG: 작업 기억은 사용자가 동의한 디자인 선택 추천에만 쓰며 기본 인계에는 넣지 않습니다.
- RBG: 코딩 AI가 생성 전에 원본 구조·규칙·진입점·검증 방법을 실제 파일에서 먼저 확인합니다.
