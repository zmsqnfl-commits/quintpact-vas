# 기존 프로젝트 가져오기

## 화면에서 하기

1. `Run-VAS-System.bat`를 실행합니다.
2. 허브에서 **기존 프로젝트 가져오기**를 누릅니다.
3. **원래 프로젝트가 있는 폴더를 선택하세요**에서 폴더를 고릅니다.
4. 분석 보고서의 기술 스택, 용량, Git, 경고를 확인합니다.
5. 새 프로젝트 이름과 보존 옵션을 확인하고 **가져오기 승인**을 누릅니다.
6. 완료 화면에서 프로젝트 열기 또는 되돌리기를 선택합니다.

## 브라우저만 사용하는 경우

브라우저의 폴더 선택 기능으로 읽기 전용 분석 보고서는 만들 수 있습니다. 브라우저 보안상 실제 복제는 할 수 없으므로 Windows 실행본에서 다시 진행합니다.

## 가져온 뒤

- 결과는 `workspace/projects/<프로젝트명>/`에 있습니다.
- 원본은 그대로 남습니다.
- Git 이력과 복구 자료는 가능한 범위에서 보존됩니다.
- RAG 연결을 선택하면 허용된 텍스트 파일만 이 기기 안에서 색인합니다. `.env`, 키·토큰, 빌드 산출물, 의존성, 절대 경로는 제외하거나 가립니다.
- VAS가 프로젝트 지식을 자동 전송하지는 않습니다. 디자인 스튜디오에서 **로컬 RAG 맥락 포함**을 선택해 복사한 프롬프트를 외부 AI에 붙여넣으면 선택된 맥락도 함께 전달될 수 있습니다.
- RAG 선택을 끄면 해당 프로젝트를 색인하지 않습니다.
- 실행·빌드 명령은 분석 결과를 확인한 뒤 사용자가 직접 승인합니다.

## 되돌리기

가져오기 결과가 맞지 않으면 완료 화면의 **가져오기 되돌리기**를 사용합니다. 원본에는 영향을 주지 않고 VAS 쪽 대상과 등록 정보만 복구합니다.
