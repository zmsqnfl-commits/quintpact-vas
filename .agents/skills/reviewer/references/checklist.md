# 코드 리뷰 체크리스트

이 디렉터리는 Reviewer 에이전트가 리뷰 시 참조하는 체크리스트를 저장합니다.
`/setup-from-application` 실행 시 프로젝트 기술 스택에 맞게 커스터마이징됩니다.

## 공통 체크리스트

### 필수 검증
- [ ] 파일당 500줄 이하 준수
- [ ] 명명 규칙(snake_case/camelCase) 일관성
- [ ] 주석 언어(한국어) 준수
- [ ] 하드코딩된 매직 넘버 없음
- [ ] 불필요한 import/require 없음

### 성능 검증
- [ ] 반복문 내 불필요한 연산 없음
- [ ] 대용량 데이터 처리 시 메모리 고려
- [ ] 파일/DB 연결 후 적절한 해제
- [ ] 불필요한 동기 블로킹 없음

### 보안 사전 검토
- [ ] 사용자 입력값 검증(Validation) 존재
- [ ] 로그에 민감 정보 노출 없음
- [ ] SQL 인젝션/XSS 가능성 없음 (해당 시)

### 접근 제어 검토
- [ ] Implementer가 RBAC 허용 경로에서만 작업했는가
- [ ] ABAC 정책 위반 여부 없음

### 디자인 명세 검토 (UI 변경 시)
- [ ] 디자인 토큰(색상, 폰트, 간격)이 Designer 명세대로 적용되었는가
- [ ] 컴포넌트 구조(이름, 속성, 상태)가 명세와 일치하는가
- [ ] 에셋 파일(`src/assets/`)이 올바르게 참조되고 있는가

> 프로젝트별 기술 스택 전용 체크리스트는 `/setup-from-application` 시 추가됩니다.
