# 보안 체크리스트 레퍼런스

이 디렉터리는 Security 에이전트가 보안 검토 시 참조하는 체크리스트를 저장합니다.
`/setup-from-application` 실행 시 프로젝트 데이터 유형에 맞게 커스터마이징됩니다.

## 공통 보안 체크리스트

### 데이터 보호
- [ ] 민감 데이터(PII)가 로그에 평문 기록되지 않는가
- [ ] 인증 정보(API 키, 비밀번호)가 코드에 하드코딩되지 않았는가
- [ ] `.env` 파일이 `.gitignore`에 포함되어 있는가
- [ ] 민감 데이터 전송 시 암호화(HTTPS, TLS) 사용

### 입력 검증
- [ ] 외부 입력값에 대한 유효성 검증(Validation) 수행
- [ ] SQL 인젝션, XSS, CSRF 방어 (해당 시)
- [ ] 파일 업로드 시 확장자/크기 제한

### 리소스 관리
- [ ] 파일/DB 연결/소켓 사용 후 적절히 해제
- [ ] 메모리 집약적 작업 후 명시적 정리 (영상/음성 처리 시)
- [ ] 임시 파일 사용 후 삭제

### ABAC 정책 검증
- [ ] `sensitive_data_guard` 대상 파일이 적절히 보호되는가
- [ ] `prod_config_lock` 대상 파일의 수정 권한이 올바른가
- [ ] `review_before_final` — 리뷰/테스트 통과 전 final/ 쓰기 시도 없는가

> 프로젝트별 보안 요건은 `/setup-from-application` 시 추가됩니다.
