# /migration-cycle

기존 프로그램을 VAS에 안전하게 연결하는 흐름입니다.

1. 사용자가 원래 프로젝트 폴더를 선택합니다.
2. Architect가 읽기 전용으로 구조·스택·Git·위험 요소를 분석합니다.
3. 사용자에게 분석 보고서와 복제 옵션을 보여주고 승인을 받습니다.
4. 체크포인트, SHA-256 목록, 가능한 Git bundle을 만듭니다.
5. staging으로 복제하고 링크 이탈·해시·대상 충돌을 검사합니다.
6. `workspace/projects/<name>/`으로 원자적으로 승격합니다.
7. 프로젝트 목록과 선택적 RAG 색인을 갱신합니다.
8. 단일 테스트를 실행하고 되돌리기 ID를 제공합니다.

원본 실행·수정·삭제는 기본 흐름에서 금지합니다. 원본 삭제는 별도 고급 명령과 정확한 확인 문구가 있어야 합니다.
