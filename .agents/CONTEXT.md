# VAS 2.6.0 공통 컨텍스트

## 목적

VAS는 새 프로그램 제작과 기존 프로그램의 안전한 마이그레이션을 하나의 로컬 허브에서 연결합니다. 사용자 동의가 있을 때만 로컬 RAG와 사용 기록으로 검색·추천을 개인화합니다.

## 기준

- 단일 작업 원본: `src/`, `docs/`, `scripts/`, `tests/`
- 사용자 프로젝트: `workspace/projects/` (Git·배포 제외)
- 배포 생성물: `dist/` (Git 제외)
- 구조 문서: `docs/index.md`
- 파일당 500줄 이하, 외부 CDN·원격 폰트 금지
- 새 라이브러리 추가 전 사용자 확인

## 흐름

- 새 작업: 허브 → 신청서/디자인 → `workspace/projects/<name>/`
- 기존 작업: 폴더 선택 → 읽기 전용 분석 → 승인 → 백업/복제/검증 → 등록
- 개인화: 명시적 동의 → 최소 이벤트 → 로컬 RAG → 메모리 화면에서 사용자 제어
- 릴리즈: 단일 테스트 → 브라우저 테스트 → 패키지 생성/검증

## 안전 규칙

- 기존 프로젝트 분석 중 파일을 실행하지 않습니다.
- 원본은 기본적으로 수정·삭제하지 않습니다.
- 비밀값, 파일 내용, 절대 경로, 파일명은 개인화 이벤트에 기록하지 않습니다.
- 10회 스트레스 테스트는 사용자가 명시한 경우에만 실행합니다.
- 자연어 요청도 적절한 워크플로우로 자동 연결하고 중간 승인 지점만 사용자에게 보여줍니다.

## 완료 기준

`npm.cmd run test:python`, `npm.cmd run test:browser`, `npm.cmd run test:package`가 통과하고 캐시·비밀·사용자 데이터가 Git에 없으며 `git status --short`가 비어 있어야 합니다.
