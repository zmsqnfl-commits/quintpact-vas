# [VAS 2.5 — HTML 레이어 프로젝트] — AI 에이전트 공통 컨텍스트

> 이 파일은 **매 턴마다 항상 로드**됩니다. 핵심 정보만 간결하게 유지하세요.
> 상세 레퍼런스는 `docs/INSTRUCTIONS.md`를 참조합니다.

## 프로젝트 요약
- **목적:** VAS 2.5 시스템의 **디자인 시스템 코어 및 로컬 테마 커스터마이저 허브**. 사용자가 테마와 디자인을 기획·편집·내보내기 하는 디자인 스튜디오 환경을 제공하며, NAS에 배포된 운영용 클라이언트 폼의 의존성은 최소화함.
- **기술 스택:** HTML / Vanilla CSS / Vanilla JS
- **배포 환경:** 완전 독립형 로컬 파일 (서버 불필요, Run-VAS-System.bat 또는 src/vas-hub.html 직접 실행)

## 핵심 규칙
- 파일당 **500줄 이하** 유지 (초과 시 즉시 분리)
- 명명 규칙: kebab-case (HTML 파일), camelCase (JS 함수)
- 주석 언어: 한국어
- **AI 코딩 4대 원칙 + Taste-Skill:** 뻔한 템플릿 금지, 외과수술적 정밀 수정, 목표 주도 검증 (상세: docs/INSTRUCTIONS.md)
- **Final-Centric 라이프사이클:** 모든 작업(신규/마이그레이션)의 원본은 `final/`이며, `src/`는 일회성 수술대로 사용.
- 새 라이브러리 설치 전 **반드시 사용자 확인**
- **경량 공유 우선:** 여러 사람이 바로 쓰도록 프로젝트 폴더에는 무거운 플러그인·개인 스킬·런타임을 넣지 않고, 필요한 확장은 사용자 홈에 선택 설치.
- **VAS는 제작 도구:** 사용자가 명시적으로 "VAS/허브/스튜디오 자체 수정"을 말하지 않으면 이 폴더를 제품 코드로 간주하지 말고, 새 산출물은 `final/projects/<project-name>/`에 격리 생성.
- 🤖 **자연어 워크플로우 매핑:** 사용자가 `/dev-cycle` 같은 슬래시 커맨드를 명시하지 않고 자연어로 요구하더라도, 의도를 파악해 스스로 적합한 워크플로우를 자동 가동할 것.
- ⚡ **Auto-Orchestrator Protocol:** AI는 `[인계]` 폼을 출력하고 사용자의 입력을 기다리지 마십시오. 스스로 바통을 넘겨받아 다음 에이전트 페르소나를 장착하고 즉시 다음 Step을 연속 실행(Auto-chaining)하여, 시작부터 최종 코드 저장 및 로그 업데이트까지 단 한 번의 세션으로 논스톱 완주하십시오.
- ⚡ **경량화 테스트 프로토콜:** 평소 작업 및 검증 시에는 10회 연속 스트레스 테스트 전체를 구동하지 않고, 단 1회 실행하는 **단일 경량 스모크 테스트(Single Smoke Test)** 또는 개별 유닛 테스트만 수행합니다. 무거운 10-Loop 스트레스 테스트는 오직 사용자가 명시적으로 `'스트레스 테스트 해'` 또는 `'릴리즈 테스트 해'`라고 요청했을 때에만 최종 실행합니다.

## 에이전트 인계 포맷
```
[인계] [현재 에이전트] → [다음 에이전트]
- 완료: [한 줄 요약]
- 변경/생성 파일: [파일 목록]
- [Advisor 참조] [사용 시 질문/답변 요약 / 미사용 시 생략]
- 특이사항: [주의사항 / 없으면 '없음']
```
